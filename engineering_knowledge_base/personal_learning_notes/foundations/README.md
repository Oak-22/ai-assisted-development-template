# Foundations

This README defines the purpose of the foundations section of the
engineering knowledge base.

Use this directory for reusable conceptual notes that support future
engineering work.

## What Belongs Here

- concepts repeatedly used in implementation work
- practical rules of thumb derived from incidents or reviews
- "how to think" frameworks, not one-off task logs

## Example Enterprise-Oriented Entries

- `api-versioning-and-backward-compatibility.md`
- `data-consistency-models-in-distributed-systems.md`
- `idempotency-patterns-for-retry-safe-operations.md`
- `observability-signals-metrics-logs-traces.md`

## Strong Entry Checklist

1. Defines the concept in your own words.
2. Shows where it appeared in real project work.
3. Captures failure modes and anti-patterns.
4. Includes a quick decision rubric for reuse.
